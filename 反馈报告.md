# 提示词、Tokenizer、长度设置与推理后端反馈报告

更新时间：2026-05-15

本报告基于有卡模式最终复核结果更新。已按官方文档和本地 `tokenizer_config.json` / `generation_config.json` 重新核验 HY、NiuTrans、Gemma、Qwen 系列配置，并覆盖重跑不合规范或采样参数不一致的正式结果。当前 7 个模型、4 个方向的 smoke/full 均已完成。

## 结论摘要

1. `hy_mt1_5_7b` 已使用 HY-MT 官方调用方式：`apply_chat_template`、`add_generation_prompt=false`、无默认 system prompt、官方 prompt 风格与采样参数；旧异常结果已覆盖。
2. `niutrans_lmt_60_8b` 已使用 NiuTrans LMT 官方语言标签 prompt、`add_generation_prompt=true`、`num_beams=5`、`do_sample=false`，并设置 `fix_mistral_regex=true`；旧源文复制问题已覆盖。
3. `qwen3_32b` 与 `qwen3_moe_30b_a3b` 均显式 `enable_thinking=false`，并使用 Qwen3 官方非 thinking 推荐采样参数。
4. `gemma3_27b_it`、`gemma4_31b_it`、`gemma4_moe_26b_a4b_it` 均走本地 chat template，显式 `enable_thinking=false`，并按本地/官方 generation config 使用采样参数。
5. 推理后端是 HuggingFace Transformers `AutoModelForCausalLM.generate`，不是 vLLM；COMET 使用 `Unbabel/wmt22-comet-da`，CPU 评估，两个参考分别打分后取均值，未做额外 0-1 归一化。
6. `max_new_tokens` 审计结论：HY 为 2048，其余模型为 512。缅语参考分词长度最大约 105-107 个空格分词 token，512 不像是全局低分主因；NiuTrans 的 `zh2my/en2my` 输出偏短仍需按样例继续分析。

## 官方文档核验

- HY-MT 官方仓库：https://github.com/Tencent-Hunyuan/HY-MT
- NiuTrans LMT 官方仓库：https://github.com/NiuTrans/LMT
- Qwen3 本地模型卡：`models/qwen3_32b/README.md`
- Gemma4 本地模型卡：`models/gemma4_31b_it/README.md`

### HY-MT

- 官方示例使用 `tokenizer.apply_chat_template(..., add_generation_prompt=False)`，生成上限 `max_new_tokens=2048`。
- 当前配置：`chat_add_generation_prompt: false`、`max_new_tokens: 2048`、`top_k=20`、`top_p=0.6`、`temperature=0.7`、`repetition_penalty=1.05`。
- 当前 prompt 使用官方风格：中文方向为“只输出译文”的中文模板，英文方向为 `Translate the following segment into ...`。

### NiuTrans LMT

- 官方示例使用语言标签 prompt，例如 `English: ... Chinese:`，并使用 `max_new_tokens=512,num_beams=5,do_sample=False`。
- 当前配置：`uses_chat_template: true`、`padding_side: left`、`tokenizer_kwargs.fix_mistral_regex: true`、`max_new_tokens: 512`、`num_beams: 5`。
- 复测后 `my2zh/my2en` 源文复制问题已消失；`zh2my/en2my` 分数仍低，主要表现为输出偏短和译文质量不足。

### Qwen3 / Qwen3-MoE

- Qwen3 模型卡明确支持 `enable_thinking=False`，并建议非 thinking 模式使用 `temperature=0.7,top_p=0.8,top_k=20,min_p=0`。
- 当前 `qwen3_32b` 和 `qwen3_moe_30b_a3b` 均显式设置 `enable_thinking: false`，未使用 thinking 模式。
- 本地输出未检出 `<think>`、`</think>` 等 thinking 标记。

### Gemma3 / Gemma4 / Gemma4-MoE

- Gemma4 模型卡示例使用 `apply_chat_template(..., add_generation_prompt=True, enable_thinking=False)`，推荐 `temperature=1.0,top_p=0.95,top_k=64`。
- Gemma3/Gemma4/Gemma4-MoE 当前均显式设置 `enable_thinking: false`，使用 chat template，并将 `max_new_tokens` 提升到 512。
- Gemma3、Gemma4、Gemma4-MoE 已使用官方/本地 generation config 采样参数重跑，summary 已更新为最终结果。

## 后续建议

1. 如果要继续解释 NiuTrans `zh2my/en2my` 偏低，应抽样检查 raw 输出、分词后输出、参考分词和 EOS/早停，而不是优先继续改 prompt。
2. 若要提速，可新增 vLLM backend 并单独做等价性 smoke；不建议把后端切换和模型性能结果混在同一组正式实验中。
3. COMET 当前固定 CPU 评估较慢，可后续给 `eval_comet.py` 增加 `--gpus` 与 `--batch-size` 参数；这不影响当前已生成的翻译结果。
